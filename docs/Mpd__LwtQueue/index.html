Module Mpd__LwtQueue

Adds the file URI to the playlist (directories add recursively). URI can also be a single file.

val clear : Mpd.LwtClient.c ‑> Mpd.Protocol.response Lwt.t

Clears the current playlist.

val delete : Mpd.LwtClient.c ‑> int ‑> ?⁠position_end:int ‑> unit ‑> Mpd.Protocol.response Lwt.t

Deletes a song or a set of songs from the playlist. The song or the range of songs are identified by the position in the playlist.

val deleteid : Mpd.LwtClient.c ‑> int ‑> Mpd.Protocol.response Lwt.t

Deletes the song SONGID from the playlist.

val move : Mpd.LwtClient.c ‑> int ‑> ?⁠position_end:int ‑> int ‑> unit ‑> Mpd.Protocol.response Lwt.t

Moves the song at FROM or range of songs at START:END to TO in the playlist.

val moveid : Mpd.LwtClient.c ‑> int ‑> int ‑> Mpd.Protocol.response Lwt.t

Moves the song with FROM (songid) to TO (playlist index) in the playlist. If TO is negative, it is relative to the current song in the playlist (if there is one).

val playlist : Mpd.LwtClient.c ‑> p Lwt.t

Get a list of Song.s that represents all the songs in the current playlist.

val playlistid : Mpd.LwtClient.c ‑> int ‑> p Lwt.t

Get a list with the Song.s of the song id in the playlist

val playlistfind : Mpd.LwtClient.c ‑> string ‑> string ‑> p Lwt.t

Finds songs in the current playlist with strict matching.

val playlistsearch : Mpd.LwtClient.c ‑> string ‑> string ‑> p Lwt.t

Searches case-insensitively for partial matches in the current playlist.

val swap : Mpd.LwtClient.c ‑> int ‑> int ‑> Mpd.Protocol.response Lwt.t

Swaps the positions of SONG1 and SONG2.

val shuffle : Mpd.LwtClient.c ‑> ?⁠range:(int * int) ‑> unit ‑> Mpd.Protocol.response Lwt.t

Shuffles the current playlist. START:END is optional and specifies a range of songs.

val prio : Mpd.LwtClient.c ‑> int ‑> ?⁠range:(int * int) ‑> unit ‑> Mpd.Protocol.response Lwt.t

Set the priority of the specified songs. A higher priority means that it will be played first when "random" mode is enabled. A priority is an integer between 0 and 255. The default priority of new songs is 0.

val prioid : Mpd.LwtClient.c ‑> int ‑> int list ‑> Mpd.Protocol.response Lwt.t

Same as prio, but address the songs with their id.

val swapid : Mpd.LwtClient.c ‑> int ‑> int ‑> Mpd.Protocol.response Lwt.t

Swaps the positions of SONG1 and SONG2 (both song ids).

val rangeid : Mpd.LwtClient.c ‑> int ‑> ?⁠range:(float * float) ‑> unit ‑> Mpd.Protocol.response Lwt.t

Specifies the portion of the song that shall be played. START and END are offsets in seconds (fractional seconds allowed); both are optional. Omitting both (i.e. sending just ":") means "remove the range, play everything". A song that is currently playing cannot be manipulated this way.

val cleartagid : Mpd.LwtClient.c ‑> int ‑> string ‑> Mpd.Protocol.response Lwt.t

Removes tags from the specified song. If TAG is not specified, then all tag values will be removed. Editing song tags is only possible for remote songs.
